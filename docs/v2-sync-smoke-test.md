# v2 Sync Smoke Test Checklist

> 目標：快速驗證 v2 的共享登入、購物/行程資料同步與相片流程是否可用。

## 測試前置
- 準備兩台裝置（或兩個瀏覽器 Profile）以同一組共享帳戶登入。
- 兩邊都確認可正常讀取 Supabase（非匿名共享模式）。

## 1) Shared Login（共享登入）
- [ ] 裝置 A 以 Email/密碼登入。
- [ ] 裝置 B 使用同一組帳戶登入。
- [ ] 兩邊「設定 > 同步帳戶」都顯示已登入同一帳戶。
- [ ] 登出後重新登入，資料仍可正確讀取。

## 2) Shopping Sync（購物清單同步）
- [ ] 裝置 A 新增 1 筆購物項目（含名稱、分類）。
- [ ] 裝置 B 在前景/重新聚焦後可看到該項目。
- [ ] 裝置 B 編輯該項目（例如備註/價格），裝置 A 可同步看到更新。
- [ ] 裝置 A 勾選完成狀態，裝置 B 狀態一致。

## 3) Shopping Photo（購物相片）
- [ ] 在裝置 A 編輯購物項目，新增或更換相片。
- [ ] 按「儲存」後，相片可於 A 顯示，且 B 重新聚焦後可看到相片。
- [ ] 再次編輯改為移除相片並儲存，A/B 皆不再顯示該相片。

## 4) Itinerary Sync（行程/活動同步）
- [ ] 裝置 A 新增 1 筆活動（指定日期、時間、地點）。
- [ ] 裝置 B 可同步看到該活動。
- [ ] 在任一裝置調整活動排序（上移/下移），另一裝置排序一致。
- [ ] 刪除活動後，另一裝置也同步移除。

## 5) Activity Photo（活動相片）
- [ ] 在活動編輯面板新增相片，按「儲存」。
- [ ] 重新開啟活動可看到相片，且第二裝置也可看到。
- [ ] 移除相片後按「儲存」，兩裝置都不再顯示。

## 6) Second-device Verification（第二裝置驗證）
- [ ] A 連續做 2~3 次異動（新增、編輯、刪除混合）。
- [ ] B 每次在切回前景後都能看到最終狀態。
- [ ] B 的異動也可反向同步回 A。

## 7) Save / Cancel Photo Behavior（相片儲存/取消行為）
- [ ] 在編輯面板選擇新相片後按「取消」關閉：不應改動既有相片資料。
- [ ] 重新進入編輯面板，確認前一次未儲存的相片草稿不應殘留。
- [ ] 選擇「移除相片」後按「取消」：原相片仍應保留。
- [ ] 只有在按「儲存」時，新增/更換/移除相片才會真正寫入雲端與本機狀態。

## 補充觀察（可選）
- [ ] 若雲端同步失敗，畫面顯示明確提示，且本機資料仍保留。
- [ ] 在 production 不出現過量 sync debug logs；preview/development 可看到必要診斷資訊。
